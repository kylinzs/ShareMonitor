# indices-sectors Specification

## Purpose
定义“行情”页面的指数与板块能力，包括列表展示、排序、板块详情与加入自选、刷新间隔与错误处理，并约束 MVP 下行情页数据源固定为东方财富。
## Requirements
### Requirement: 应用提供“行情”入口
系统应当（SHALL）在主导航中提供一个可见的入口进入“行情”页面。

#### Scenario: 用户打开行情页
- **WHEN** 用户点击“行情”入口
- **THEN** 系统导航到行情页面

### Requirement: 指数列表展示关键行情字段
当可用时，系统应当（SHALL）展示主要指数列表及关键行情字段，包括 `name`、`code`、`lastPrice`、`change`、`changePct` 与 `quoteTime`。

#### Scenario: 指数列表渲染
- **WHEN** 指数行情数据可用
- **THEN** 指数列表为每个指数展示所需字段与最后更新时间

### Requirement: 板块列表可用且支持排序
系统应当（SHALL）展示板块列表，并且应当（SHALL）支持至少一种排序方式（例如按 `changePct`）。

#### Scenario: 用户按涨跌幅查看板块排序
- **WHEN** 用户选择“涨跌幅”排序模式
- **THEN** 系统按 `changePct` 对板块进行排序展示

### Requirement: 板块详情展示成分股
系统应当（SHALL）提供板块详情页，展示该板块的成分股及关键行情字段。

#### Scenario: 打开板块并查看成分股
- **WHEN** 用户从板块列表选择某个板块
- **THEN** 系统展示该板块的成分股列表

### Requirement: 成分股可加入自选且防重复
系统应当（SHALL）允许用户从板块详情页将成分股加入自选列表，并且必须（MUST）防止重复。

#### Scenario: 将成分股加入自选
- **WHEN** 用户对一个当前不在自选中的成分股点击“Add”（添加）
- **THEN** 系统将其加入自选列表并相应更新 UI 状态

#### Scenario: 添加已存在成分股不产生重复
- **WHEN** 用户对一个已在自选中的成分股点击“Add”（添加）
- **THEN** 系统不创建重复项，并展示非阻塞的确认提示

### Requirement: 指数/板块刷新遵守最小间隔
系统必须（MUST）对指数、板块以及成分股行情刷新执行配置的最小刷新间隔（默认 15 分钟）。

#### Scenario: 间隔内刷新使用缓存数据
- **WHEN** 用户在距离上次成功更新仍处于配置刷新间隔内触发刷新
- **THEN** 系统不发起网络请求，并继续展示缓存数据，同时提供用户可见的提示

### Requirement: 指数/板块加载失败对用户可见
如果指数/板块数据加载失败，系统必须（MUST）告知用户并提供重试操作。

#### Scenario: 加载失败并重试
- **WHEN** 加载指数或板块失败且用户点击重试
- **THEN** 系统再次尝试加载，并在成功时更新 UI

### Requirement: MVP 下行情页使用东方财富（Eastmoney）数据源
在 MVP 中，系统必须（MUST）使用东方财富（Eastmoney）作为“行情”页面的数据源，而不受所选行情提供方影响。

#### Scenario: 切换行情提供方不影响行情页数据源
- **WHEN** 用户将行情提供方切换为腾讯
- **THEN** 行情页仍然从东方财富（Eastmoney）加载指数与板块数据
