# settings Specification

## Purpose
定义设置页的行情数据源选择、自动刷新配置与免责声明展示，并规定“行情”页面在 MVP 中的数据源固定为东方财富。
## Requirements
### Requirement: 用户可选择行情数据源
系统应当（SHALL）提供设置项用于在多个行情数据源之间选择，并提供默认选择。默认数据源必须（MUST）为“东方财富”。

#### Scenario: 默认数据源被选中
- **WHEN** 用户首次打开设置
- **THEN** 系统展示默认数据源选择为“东方财富”

#### Scenario: 数据源切换可持久化
- **WHEN** 用户修改数据源选择
- **THEN** 系统持久化该选择，并将其用于后续行情请求

#### Scenario: 用户可在东方财富与腾讯间选择
- **WHEN** 用户打开数据源选择
- **THEN** 当两者均受支持时，UI 至少展示“东方财富”和“腾讯”作为可选项

### Requirement: 数据源切换对功能生效范围一致
系统必须（MUST）将所选行情提供方应用到自选列表行情与个股详情（行情 + 历史）中。在 MVP 中，“行情”（indices/sectors）页面必须（MUST）始终使用东方财富（Eastmoney）作为数据源。

#### Scenario: 切换数据源影响后续加载
- **WHEN** 用户切换数据源后再进入其它页面或触发加载
- **THEN** 后续数据加载使用新选择的数据源

### Requirement: 仅展示支持的行情数据源
系统应当（SHALL）仅将满足 MVP 行情能力集的数据源作为可选项展示：支持自选列表行情与个股详情历史（Intraday + DailyK）。

#### Scenario: 不支持的数据源不展示
- **WHEN** 某数据源缺失所需能力
- **THEN** 设置页不将其展示为可选项

### Requirement: 用户可控制自动刷新行为
系统应当（SHALL）提供设置项用于启用或禁用自动刷新，并且应当（SHALL）允许从预设集合中配置自动刷新间隔。默认间隔必须（MUST）为 15 分钟。

#### Scenario: 自动刷新被禁用
- **WHEN** 用户禁用自动刷新
- **THEN** 系统不会在没有用户明确操作的情况下周期性拉取行情

#### Scenario: 自动刷新间隔生效
- **WHEN** 用户启用自动刷新并选择某个间隔
- **THEN** 当自选列表可见时，系统按不高于该间隔的频率刷新行情

### Requirement: 设置包含数据免责声明
系统必须（MUST）提供数据免责声明，并且必须（MUST）展示当前选择的数据源名称。

#### Scenario: 免责声明可见
- **WHEN** 用户打开设置页
- **THEN** UI 展示免责声明与所选数据源名称
